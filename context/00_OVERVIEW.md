# 教材纲领：分布式系统入门

## 受众

有编程经验的开发者（后端、全栈、基础架构方向），写过多节点部署的服务或用过数据库/缓存/消息队列，但没有系统学过分布式系统理论。

可以假设读者具备：
- 基本编程能力（变量、函数、控制流、数据结构）
- 后端开发经验（写过 Web 服务、用过数据库、部署过应用）
- 对网络通信的朴素理解（HTTP 请求、客户端-服务端模型）
- 常见工具的使用（编辑器、命令行、git）

不可以假设读者了解：
- 分布式系统理论（CAP、FLP、一致性模型、共识协议）
- 数据库内部实现（存储引擎、事务实现、复制协议）
- 分布式算法（Paxos、Raft、向量时钟）
- 形式化方法（I/O 自动机、TLA+）
- 底层网络细节（TCP 的具体实现、网络栈）

## 学习目标

读者学完后应能：

1. **识别分布式问题**：在日常开发中遇到缓存不一致、消息重复、超时困境等现象时，能识别出背后的分布式系统问题并给出分类
2. **理解核心权衡**：能解释为什么分布式系统中不存在"既一致又可用又分区容错"的完美方案，并在具体场景中判断应优先保证什么
3. **推导设计决策**：面对一个需要复制、分区或事务保证的系统设计问题，能从问题出发推导出合理的方案，而非套用模板
4. **评估现有系统**：能读懂 Redis、Kafka、Zookeeper 等常用中间件的一致性保证和故障行为，在架构选型时做出知情决策
5. **实现核心机制**：能用代码实现简化版的复制、故障检测、共识等机制，理解这些机制在中间件内部如何工作

## 边界

### 覆盖
- 分布式系统的核心问题和约束（网络不可靠、无全局时钟、部分故障）
- 复制、分区、一致性模型、共识协议的完整推导
- 分布式事务、故障检测、时间与顺序
- 上述概念在日常开发中的映射（缓存、消息队列、微服务通信、分布式锁）

### 不覆盖（及理由）
- 特定中间件的运维和调优（这是操作手册的职责，不是入门教材的职责）
- 分布式机器学习和 AI 系统（专门领域，超出入门范围）
- 区块链和拜占庭容错的深入展开（入门阶段聚焦崩溃故障模型即可）
- 形式化验证方法（TLA+等，属于进阶内容）
- 特定编程语言的分布式框架（如 Akka、Erlang/OTP 的深入使用）

## 教学策略

**混合型：症状驱动 + 推导链 + 渐进式实践**

### 1. 制造问题，而非抛出问题
不直接告诉读者"分布式系统存在一致性问题"，而是带读者从一个朴素方案出发，逐步引入真实约束（网络延迟、节点崩溃、时钟不同步），让问题在读者眼前自然浮现。读者只有捋清楚了问题产生的原理，才能真正理解解决方案，以及"如何思考解决方案"。这是"动机先行"的加强版——动机不是被告知的，而是被体验的。

### 2. 症状驱动的入口
不从抽象定义开始，从程序员日常遇到的分布式"症状"出发，逆向推导出底层机制。这是本教材的核心差异点——解决"没有抓手"的问题。

### 3. 推导链驱动的理论 + 难点追踪
每个概念遵循"问题 -> 约束 -> 方案 -> 合理性"的推导路径（借鉴 DDIA 的优势）。在一致性和共识这两个公认的教学难点处，增加中间推导节点的密度，不跳步。

对每一课，在教研记录中有意识地标注难点，记录格式：
- **表现**：读者在这里可能卡住，因为……
- **推导链缺失猜测**：我们认为读者可能缺少的前置节点是……
- **应对**：我们通过以下方式补全缺失——……
- **验证标准**：如果读者反馈"……"说明猜对了；如果反馈"……"说明缺失在别处

这些记录的价值不在于"猜对了"，而在于"知道自己是怎么猜的"，为后续复盘提供依据。当同类缺失反复出现时，上升为受众假设的修正。

### 4. 暴露被封装的机制
通过渐进式构建，让读者亲手实现被中间件藏起来的分布式机制。把"不可观测"变成"可观测"——让读者能像调试单机程序一样，看到分布式系统中事件的因果关系和故障的传播路径。

### 5. 真实世界的练习
不用每一课都有练习，但每次练习必须是一个完整的、连接真实世界的实践。关键词是**真实世界**。

具体原则：
- 练习场景来自日常开发中的高频问题（缓存一致性、消息语义、复制故障），不是为教学编造的
- 完成后读者能对照 Redis、Kafka、Zookeeper 等真实系统理解其内部机制
- 一次练习就是一个完整的、有价值的实践，让读者感到收获满满
- 宁缺毋滥——没有自然实践锚点的课不硬塞练习，等理论积累足够后设计综合练习一次性激活多个概念
- 避免无意义和枯燥的练习（孤立的函数实现、与分布式无关的 TCP echo server 等）

### 6. 理论课与项目课交织
理论课提供知识前置，项目课在理论课之后提供实践锚点。项目课只使用已教过的概念，明确标注理论前置。

## 调研基础

本纲领基于以下调研形成（详见 `data/research/`）：
- 宏观扫描 7 本教科书、4 门公开课、3 个开源教程/项目、4 个中文资源
- 深入分析 DDIA、MIT 6.824、Tanenbaum 三本代表性教材的教学法（7 维度拆解）
- 验证"分布式系统缺乏抓手"假设（对比其他学科、搜集程序员痛点、评估已有实践导向尝试）
